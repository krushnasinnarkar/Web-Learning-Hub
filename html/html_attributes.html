What is HTML Attribute?

  • All elements can have attributes. And every attributes containes additional pieces of information about tags.
  • We can also say that attributes are the spesial words that can modifie the elements.
  • An attribute used to define the characteristics of an HTML element.
  • Attribute should always be specified in start tag of HTML Element.
  • All attributes are made up of two parts −
    1. Attribute Name
    2. Attribute Value

Syntax:

<tag name attribute_name="attribute_value"> Some Content </tag name>

Example

<h1 style="color: blueviolet"> This is heading with blueviolet text color. </h1>
<p style="background-color: blue"> This is paragraph with blue background. </p> Output
RK Editor

Output :

This is heading with blueviolet text color.

This is paragraph with blue background.

The style attribute is used to add styles to an element, such as text color, background color, text alignment, text size and more.

Tip : You can learn much more about styles in our HTML Style Attribute chapter.

Boolean Attributes

There are several attributes in HTML5 that are consists of just name or its value must either be the empty string or a value that is an ASCII case-insensitive match for the attribute's canonical name, with no whitespace. Such attributes are called Boolean attributes.

The presence of a boolean attribute on an element represents the true value, and the absence of the attribute represents the false value.

The values "true" and "false" are not allowed on boolean attributes. To represent a false value, the attribute has to be omitted altogether.

This means that <div hidden="true"> is not allowed in HTML5.
Correct would be either <div hidden> or <div hidden=""> or <div hidden="hidden"> or case-insensitive and single quotes/unquoted variations of any of them.

<!--Attribute with just name-->
<div hidden>This is valid HTML but invalid XML.</div>
<!--Attribute value as empty string-->
<div hidden="">This is valid HTML and also valid XML.</div>
<!--Attribute value that is an ASCII case-insensitive match for the attribute's canonical name-->
<div hidden="hidden">This is also valid HTML and XML</div>
<div hidden=hidden>This is also valid HTML but invalid XML.</div>

Guide lines for Attributes

1. Always use lowercase attributes

HTML Attributes are not case sensitive. Means all attributes can be written with lowercase or uppercase like href or HREF.

However, W3C recommended to use lowercase attributes in HTML. And for stricter document types like XHTML lowercase attributes are strickly demanded.

2. Always Quote Attribute Values

According to the HTML standard attribute values does not require quotes around it.

However, W3C recommended that always quote attribute values. And for stricter document types like XHTML quote attribute values are always required.

Good:

<p title="RK Fundamentals">At RK Fundamentals we always use quotes around attribute values.</p>

Bad:

<p title=RK Fundamentals>This example will not display the title attribute correctly, because it contains a space.</p>

Double quotes around attribute values are the most common in HTML, but some people might use single quotes in HTML. So use of single quotes with HTML attribute, is also allowed.

In some situations, when the attribute value itself contains double quotes, it is necessary to use single quotes:

<p title="This is 'Paragraph' tag"></p>

Or vice versa:

<p title='This is "Paragraph" tag'></p>

3. widht and height Attributes

The <img> tag should also contain the widht and height attributes. This attributes are used to specifie the width and height of an image(or any other tag). Values of this attributes are numericals of pixals(px), or a percentage(%) of the page or parent element. Default values of this attributes are in pixals(px).

If values are in percentage(%) then one of them is considered as percentage value(by defalt width) and other as auto. but this doesn't done with pixal(px) values.

Example

<img src="demo.jpg" width="100" height="100">
<img src="demo.jpg" width="100" height="200">
<img src="demo.jpg" width="20%" height="20%">
<img src="demo.jpg" width="20%" height="50%">

<p title="Paragraph with title attribute.">Move the cursor over this text to see the value of title attribute as a tooltip.</p> Output

5. id Attribute

The id attribute is used to give an unique identification to an element within a web page. The id attribute of an element must be unique within a web page. No two elements have the same id and each element can have only one id. You can use the id attribute to distinguish between elements that have the same name.

Example

<h1 id="heading">RK Fundamentals.</h1>
<p id="info">Let's learn HTML with RK Fundamentals</p>

6. class Attribute

The class attribute specifies the class of element. Like id attribute, the class attribute is also used to identify elements. But the class attribute does not have to be unique in the web page like id attribute. This means you can apply same class to multiple elements. Also you can apply multiple classes to single element.

Syntax for applying multiple classes to single element:

When you have to apply multiple classes to single element then the value of the attribute is written in space-separated list of class names.

<tag name class=" classname1 classname2 classname3 ..."></tag name>

Example

<div class="header block">RK Fundamentals.</div>
<p class="block">Let's learn HTML with RK Fundamentals</p>
 Output

RK Fundamentals.

7. lang Attribute

The lang attribute declare the main language used in a web page. Declaring a language is can be important for assist search engines and browsers.

The values of the lang attribute are ISO-639 standard two-character language codes. Check HTML Language Codes: ISO 639 for a complete list of language codes.

<!DOCTYPE html>
<html lang="en">
<body>
...
</body>
</html>

The country codes can also be added to the language code in the lang attribute. So, the first two characters define the language of the web page, and the last two characters define the country.

Check HTML Country Codes for a complete list of country codes.

<!DOCTYPE html>
<html lang="en-IN">
<body>
...
</body>
</html>
